Introduction

This is a 1st year undergraduate course for geoscientists taught by Gerard Gorman at Imperial College London. The course has two objectives:

  • Learn the basic principles of computer programming. This understanding will form the basis for learning many other programming languages.
  • Become proficient at Python programming as it is currently one of the most commonly used languages in computational science.

The lectures notes are written in IPython notebook format. To take a quick view of the individual lectures you can simple click on the links below (uses the online tool http://nbviewer.ipython.org).

You can use Python/IPython on a wide range of computing platforms including Windows, Linux and Mac. If you have your own laptop that you want to use then ask for assistance in the lecture to get everything setup.

In the lecture we are going to run everything off the dept student linux servers - student.ese.ic.ac.uk and student2.ese.ic.ac.uk. In the first lecture you will be shown how to ssh to this server. Once you are logged in you will want to get a copy of the lecture material. To do this you execute the following command at the terminal:

git clone https://github.com/ggorman/Introduction-to-programming-for-geoscientists.git

You only need to clone the repository once in the first tutorial!

If you type ls you will see that this has created a directory Introduction-to-programming-for-geoscientists/.

Navigate to the directory containing the course material using the command:

cd Introduction-to-programming-for-geoscientists/notebook

Convince yourself that all your previous work has been saved:

git status

This should not list any files as modified. If it does, follow the steps for Saving your changes. If Git tells you about any conflicted files, call a demonstrator to help you!

The lecture material will be updated throughout the term to add exercises and sample solutions to previous weeks. So you should pull updates regularly by navigating to that directory and issuing the command:

git pull

To get started with ipython notebook issue the following command:

ipython notebook

google-chrome should start up and you should see the list of lectures. Start by clicking on lecture 0.

Remote access from your laptop

From a Mac

If you don't have an X.Org X Window System running on your Mac, install XQuartz and reboot.

Start the Terminal app or iTerm and connect to the student server with X11 forwarding, which means any graphical windows will be launched on your Mac:

ssh -Y <user>@student.ese.ic.ac.uk

Replace <user> with your username and login with your password. This gets you a terminal running on the remote server, which you can use in the same way as in the tutorial i.e. following the instructions above.

From Windows

Install PuTTY and the public domain release of XMing.

Start the XMing server, start PuTTY and type student.ese.ic.ac.uk in the Host Name field:

Make sure the option Enable X11 forwarding is ticked:

Log in with your username and password and you'll get a terminal, which you can use in the same way as in the tutorial i.e. following the instructions above.

Saving your changes

Before you leave for the day, make sure all your changes are safely stored (committed) to your local Git repository:

Check the status of your repository. This lists all the files you modified locally - convince yourself you saved all your notebooks:

git status

Mark all your notebooks as "staged for commit" such that they will be included in the next commit:

git add *.ipynb

Commit all your changes with the log message "Solutions for tutorial 5" (change that message as appropriate and do not forget the quotes!):

git commit -m "Solutions for tutorial 5"
